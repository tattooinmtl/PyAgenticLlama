Overview

What is a Skill?

A skill is two things stored together: a JSON definition that describes what the skill does and what parameters it accepts, and a Python function called execute(**kwargs) that runs the actual logic.

When you send a message, the AI reads all enabled skill definitions and decides whether to call one. If it does, PyAgenticLlama runs your Python code and feeds the result back into the conversation — all automatically.

JSON Definition

Tells the AI what the skill does, what parameters to pass, and whether it's enabled.

  • Name & description
  • Parameter schema (names, types, required)
  • Enabled / disabled flag
Python Code

Does the actual work when the AI calls the skill. Must define execute(**kwargs) and return a string.

  • Any standard library import
  • httpx, json, os pre-injected
  • Must return a str
💡
Auto-discovery If you drop a .py file with an execute(**kwargs) function into the app/skills/ folder, PyAgenticLlama registers it automatically on the next startup — no JSON file needed.
Step-by-step

Creating a Skill

1

Open the Skills panel

In the right-hand panel, click the Skills tab. Then click + New Skill at the top.

✅
The skill modal opens with empty fields and a code editor ready to type in.
2

Fill in the definition

Give your skill a name, a clear description (the AI reads this to decide when to use the skill), and add any parameters it needs.

Field What to write Example
Name Short, human-readable label Web Search
Description Explain when and how the AI should call this skill Search the web for current information on a topic
Parameters Each input the AI will fill in (name, type, required) query · string · required
Action type Almost always Python python
⚠️
Write a good description The description is what the AI sees — not you. Be specific: "Search the web using DuckDuckGo and return the top 5 results" works much better than "search".
3

Write the Python code

In the code editor, write your execute(**kwargs) function. Parameters defined in the schema are available as keyword arguments.

Python — skill template 
# Any imports you need
import httpx, json

def execute(**kwargs):
    # Read parameters — match names to your schema
    query = kwargs.get('query', '')

    # Do the work
    if not query:
        return 'No query provided.'

    # Always return a string — the AI reads this as the tool result
    return f'Result for: {query}'

What's pre-injected

You don't need to import these — they're already available inside execute():

NameWhat it is
httpxHTTP client — httpx.get(url), httpx.post(url, json={...})
jsonStandard JSON encoder/decoder
osOS utilities — environment variables, paths
subprocessRun shell commands
Pathpathlib.Path — filesystem operations
vault_getvault_get(key) — read a secret from the Vault
4

Save the skill

Click Save. The skill appears in the Skills list immediately. Toggle the switch to enable or disable it without deleting it.

✅
Enabled skills are automatically sent to the model as OpenAI-format tool definitions on every chat request.
Testing

Testing a Skill

You don't need to prompt the AI to test a skill — you can run it directly from the Skills panel.

1

Open the Test modal

In the Skills panel, click the ⚙ Test button on any skill card. The modal opens with auto-filled input fields for every parameter defined in the schema.

2

Fill in test values

Each parameter gets its own input box. Type a test value and click Run Skill.

💡
Type hints shown The modal shows the expected type (string, integer, boolean) next to each field so you know what to enter.
3

Read the result

The skill runs on the server and the output appears in the modal. If there's an error, the full Python traceback is shown — fix the code and run again.

⚠️
Skill code runs on the server Network requests, file reads, and subprocess calls all happen on the machine running PyAgenticLlama, not in your browser.

Common errors and fixes

ErrorLikely causeFix
No execute() function Function name typo Rename to exactly def execute(**kwargs):
KeyError / None Parameter name mismatch Check that kwargs.get('name') matches the schema field name exactly
Module not found Third-party package not installed Run pip install package-name in the terminal tab, then retry
Returns None / empty Missing return statement or returns non-string Always end with return str(result)
